⚑ Critical discovery during development

USART2_BRR is at offset 0x0C from USART2 base — absolute address 0x4000440C. Address 0x40004408 is CR3. Writing the baud rate divisor to CR3 silently corrupts UART configuration with no error output. Always verify register offsets against RM0351.

Hardware

System specification

MCU STM32L476RGT6
Core ARM Cortex-M4 @ 80MHz
Flash / RAM 1MB / 128KB
Clock (current) MSI default 4MHz
Sensor 1 MAX30102 — Heart Rate + SpO2
Sensor 2 TMP117 — ±0.1°C temperature
Sensor bus I2C1 — PB8 (SCL) / PB9 (SDA)
UART USART2 — PA2 TX — 9600 baud
HAL None — register-level only
Dynamic allocation None — no malloc anywhere

Design

Three-layer architecture

Prototype phase: monolithic main.c. Refactor phase: split into layers below. Each layer communicates only with the layer directly beneath it.

3 Application Main loop, alert thresholds, UART output. No register access. main.c
↓ calls
2 Processing Moving average filter, circular buffer, threshold detection. filter.c / filter.h
↓ calls
1 Driver Register writes only. UART, I2C, TMP117, MAX30102 drivers. uart.c i2c.c tmp117.c max30102.c
